## Overview

Combines real-time speech processing, VTube Studio integration, and desktop automation for autonomous streaming. Features Twitter scheduling, YouTube chat interaction, and extensible tool framework designed for 24/7 autonomous operation.

This project demonstrates the convergence of AI, streaming technology, and desktop automation to create a fully autonomous content creation system.

## Key Features

### Real-Time STT/TTS Processing
- **Groq Whisper** for ultra-fast speech-to-text (<200ms latency)
- **ElevenLabs** for high-quality text-to-speech with emotion
- Voice activity detection (VAD) for natural conversation flow
- Background noise suppression
- Multi-speaker diarization support

### VTube Studio Integration
- Real-time avatar control via WebSocket API
- Expression mapping from sentiment analysis
- Gesture synchronization with speech
- Scene management and transitions
- Custom hotkey bindings for reactions

### OBS Integration
- Automated scene switching based on activity
- Source visibility control
- Filter management (blur, color correction)
- Recording and streaming state management
- Virtual camera output

### Twitter Automation
- Scheduled tweet posting
- Thread creation and management
- Media upload (images, GIFs)
- Engagement tracking
- Reply automation with sentiment awareness

### YouTube Chat Interaction
- Real-time chat monitoring
- Sentiment analysis for responses
- Super chat highlighting
- Command processing (!info, !help)
- Moderation automation

### GraphRAG Memory System
- Long-term conversation memory
- User preference tracking
- Topic relationship modeling
- Context-aware responses
- Memory consolidation overnight

### Extensible Tool Framework
- Plugin architecture for custom tools
- Hot-reload for development
- Dependency injection
- Error recovery and retry logic
- Performance monitoring

## Tech Stack

- **Core**: Python 3.11 with asyncio
- **Speech**: Groq Whisper (STT), ElevenLabs (TTS)
- **Avatar**: VTube Studio WebSocket API
- **Streaming**: OBS WebSocket, VoiceMeeter Banana
- **Audio**: FFmpeg, PyAudio, sounddevice
- **Frontend**: TypeScript + React for control panel
- **Memory**: MongoDB with vector search

### Configuration

```yaml
# config.yaml
speech:
  stt: groq_whisper  # or openai_whisper
  tts: elevenlabs    # or google, azure
  voice_id: "rachel"  # ElevenLabs voice
  emotion_mapping: true

streaming:
  obs:
    host: localhost
    port: 4455
    password: your_password
  vtube_studio:
    host: localhost
    port: 8001

automation:
  twitter:
    enabled: true
    post_frequency: "0 */4 * * *"  # Every 4 hours
  youtube:
    chat_monitoring: true
    auto_respond: true

memory:
  provider: mongodb
  consolidation_schedule: "0 4 * * *"  # 4 AM daily
```

## Autonomous Streaming Workflow

1. **Pre-Stream**: Load schedule, check equipment, test audio
2. **Going Live**: Start OBS, activate VTube Studio, announce on Twitter
3. **During Stream**: Monitor chat, respond to viewers, execute scheduled segments
4. **Raids/Hosts**: Automatic thank you messages and reactions
5. **Post-Stream**: Save VOD highlights, post clips to Twitter, memory consolidation
6. **Overnight**: Background tasks, content planning, Twitter scheduling

## Tool Extension Example

```python
from rinai.tools import Tool, tool_registry

@tool_registry.register
class CustomTool(Tool):
    name = "custom_action"
    description = "Does something custom"

    async def execute(self, params: dict):
        # Your logic here
        result = await your_api_call(params)
        return {
            "success": True,
            "data": result
        }
```

## Challenges & Solutions

**Challenge**: Speech latency breaking immersion
**Solution**: Groq Whisper reduces STT to <200ms, pre-generating common responses

**Challenge**: Avatar movements feeling robotic
**Solution**: Sentiment-based expression mapping with randomized micro-movements

**Challenge**: Context loss in long streams
**Solution**: GraphRAG memory with rolling summarization every 30 minutes
